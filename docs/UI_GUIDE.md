# 메시지 디자인 가이드 (텔레그램)

이 프로젝트의 "UI"는 텔레그램 메시지다. 웹 UI는 없다.

## 원칙
1. 핵심(도착 N분)이 가장 먼저, 스크롤 없이 보인다.
2. 혼잡도는 글자 읽기 전에 색 점으로 판단: 🟢 여유 / 🟡 보통 / 🔴 혼잡.
3. 없는 정보는 우아하게 생략한다 (혼잡도 미제공, 정거장 수 미상 등). 빈 칸/0/None을 그대로 노출하지 않는다.

## 형식 — 노선별 카드형
시나리오당 헤더 1줄 + 노선별 카드. 각 카드는 최대 2대(다음·다다음)까지.

```
🌅 등촌역 → 당산역

🚌 6516번
   ⏱ 3분    🟢 여유   2정거장 전
   ⏱ 12분   🟡 보통   6정거장 전

🚌 660번
   ⏱ 7분    🔴 혼잡   4정거장 전
   ⏱ 21분

🕘 09:14 기준
```

- 저녁은 헤더 이모지를 🌆 로.
- 헤더의 "등촌역 → 당산역"은 stops.json의 `label`.
- 분/정거장 수를 파싱하지 못하면 원본 도착메시지(`raw_msg`)를 그대로 표시.
- 혼잡도가 None이면 해당 칸을 생략.
- 표시할 버스가 없으면 "표시할 버스가 없어요" 같은 한 줄 안내.

## 혼잡도 매핑
| 코드 | 라벨 | 이모지 |
|------|------|--------|
| 3 | 여유 | 🟢 |
| 4 | 보통 | 🟡 |
| 5 | 혼잡 | 🔴 |
| 0 / 없음 | (생략) | — |

## 금지
- HTML/마크다운 과용 금지. 플레인 텍스트 + 이모지로 충분.
- 장식용 배너/구분선 떡칠 금지. 정보 밀도 우선.
