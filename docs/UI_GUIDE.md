# 메시지 디자인 가이드 (텔레그램)

이 프로젝트의 "UI"는 텔레그램 메시지다. 웹 UI는 없다.

## 원칙
1. 핵심(도착 N분)이 가장 먼저, 스크롤 없이 보인다.
2. 혼잡도는 글자 읽기 전에 색 점으로 판단: 🟢 여유 / 🟡 보통 / 🔴 혼잡.
3. 없는 정보는 우아하게 생략한다 (혼잡도 미제공, 정거장 수 미상 등). 빈 칸/0/None을 그대로 노출하지 않는다.

## 형식 — 방면 그룹별 통합 리스트
`stops.json`의 시나리오는 `groups` 배열로 구성된다. 각 그룹은 방면 `label`(선택)과 `routes`(노선번호 목록)을 갖는다. 메시지는 헤더 1줄 + 그룹별 블록으로 구성되며, 각 그룹은 **노선당 가장 빨리 오는 버스 1대만** 도착 빠른 순(동일 분이면 혼잡도 낮은 순)으로 정렬해 최대 3개 노선까지 보여준다(`src/bot.py:_MAX_PER_GROUP`). 같은 노선이 여러 줄을 차지해 다른 노선을 밀어내지 않도록 하기 위함이다.

```
🌆 당산역 출발

🚏 등촌역 방면
🚌 6632번 · ⏱ 3분 · 🟢 여유 · 2정거장 전
🚌 605번 · ⏱ 7분 · 🔴 혼잡 · 4정거장 전
🚌 661번 · ⏱ 12분

🚏 관음·삼성아파트 방면
🚌 660번 · ⏱ 5분 · 🟡 보통 · 3정거장 전

🕘 18:14 기준
```

- 아침은 헤더 이모지를 🌅로, 저녁은 🌆로 (`build_message`의 `evening` 플래그).
- 헤더의 텍스트는 stops.json의 시나리오 `label`.
- 그룹 `label`이 빈 문자열이면 `🚏 그룹명` 줄 자체를 생략한다 (예: morning 시나리오처럼 그룹이 하나뿐일 때).
- 한 줄의 각 항목은 " · "로 구분한다 (텔레그램은 고정폭 폰트가 아니므로 공백만으로 칼럼을 맞출 수 없다).
  - 분을 파싱하지 못하면 `⏱ N분` 대신 원본 도착메시지(`raw_msg`)를 그대로 표시.
  - 혼잡도가 None이면 해당 항목을 생략.
  - 정거장 수를 파싱하지 못하면 해당 항목을 생략.
- 그룹 안에 표시할 버스가 하나도 없으면 그 그룹 블록에 "도착 예정 버스가 없어요 🚏" 한 줄만 표시.
- 시나리오 전체에 표시할 버스가 하나도 없으면 그룹 블록 없이 "표시할 버스가 없어요 🚏" 한 줄만 표시.
- 매 응답에는 "🌅 아침" / "🌆 저녁" 상시 버튼 키보드가 함께 붙는다 (`_KEYBOARD`, `src/bot.py`).
- `/start`, `/help` 명령은 이 카드 형식이 아닌 별도 도움말 텍스트(`_HELP`)를 반환한다.

## 금지
- HTML/마크다운 과용 금지. 플레인 텍스트 + 이모지로 충분.
- 장식용 배너/구분선 떡칠 금지. 정보 밀도 우선.
